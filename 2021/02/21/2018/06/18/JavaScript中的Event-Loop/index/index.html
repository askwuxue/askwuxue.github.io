JavaScript中的Event-Loop

发表于 2018-06-18 | 分类于 JavaScript

Event Loop（事件循环）

​ 对于Javascript的Event-Loop之前一直不太清除，也没有深入的学习，最近看了阮一峰老师的博客，再重新的学习一下Event-Loop。

JavaScript是单线程非阻塞的脚本语言

线程：线程是进程中的一个实体，是被系统独立调度和分派的基本单位、

单线程：JavaScript在执行任何代码的时候，都只有一个主线程在处理所有的任务。

同步任务： （synchronous）在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务；

异步任务：，（asynchronous）不进入主线程、而进入”任务队列”（task queue）的任务，只有”任务队列”通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。

非阻塞：当执行一项异步任务时，无法立即返回结果，比如I/O事件，主线程会挂起(pending)当前这个任务。然后在异步返回结果的时候再根据一定的规则去执行相应的回调

JavaScript设计之初就是设计者就想让他是单进程，历史不扯，总之，JavaScript以后还会是单线程，但是现在的web worker可以实现JavaScript的部分多线程特性，以后再说。今天的主角是JavaScript如何实现了非阻塞状态？最后回到了我们的题目Event loop（事件循环）

本文的事件循环是基于浏览器下的JS引擎的

堆和栈，不多费口舌。堆中存放对象，栈中存放基础变量以及对象指针。

我们还得提一个重要的概念，函数执行环境，（执行上下文），这个执行环境存放和这个函数有关的参数，当这些方法被调用的时候，JavaScript是单线程的，同一时间只能执行一个函数，这些方法被放在一个单独的地方，执行栈，不等同于我们说的栈。

开始执行脚本的时候，js引擎解析这段代码，将同步的代码按顺序加载到执行栈中，从头开始执行，如果执行到了方法，那么，就把这个方法的执行换件添加到执行栈中。然后继续执行这个执行环境中的代码，当前执行环境中的代码执行结束，退出执行环境并把执行环境销毁，回到上一个执行环境。这个过程反复执行，直到执行栈中的代码全部执行完毕。

盗图，形象

这个过程有点类似是剥洋葱，一层层往下剥，剥的越深，剥下来的洋葱皮就越多，直到你的小篮子装不下洋葱皮，我们所说的栈溢出。

主角出场

​ 刚才我们说，执行同步代码的时候，就是不断的产生新的执行环境并添加到执行栈，然后退出，销毁执行环境，这样的一个过程。现在，js引擎遇到的是异步事件的时候，不会一直等待返回数据，此时，将这个事件挂起，继续执行执行栈中的其他的任务，当异步事件返回结果。JS将这个事件加入与执行栈不同的事件队列中。被放入事件队列不会立即执行器回调，而是等待执行栈中的所有任务都结束，主线程闲了，然后再去事件队列看看有没有任务，如果有，取出排名第一的事件，然后把这个事件对应的回调函数加载进执行栈中，然后执行其中的代码。如此形成一个无限的循环，这个过程称之为 事件循环
